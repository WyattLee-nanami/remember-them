# 增量记忆合并器（缅怀版）

## 任务

当用户追加新的原材料或回忆时，将新内容与现有记忆合并，不覆盖已有内容，只追加和完善。

## 合并原则

1. **不覆盖，只追加**：已有记忆保持不变，新内容追加到对应节
2. **去重**：如果新内容与已有内容重复，跳过
3. **补充细节**：如果新内容是对已有记忆的补充，用 `Edit` 工具在原文后追加细节
4. **标记来源**：新追加的内容标注 `[追加于 {日期}]`

## 合并流程

1. 读取现有 `memory.md` 和 `persona.md`
2. 分析新内容属于哪个维度
3. 判断是新增还是补充：
   - **新增**：追加到对应节的末尾
   - **补充**：在已有条目后追加细节
4. 更新 `meta.json` 的 `updated_at` 和 `memory_sources`
5. 重新生成 `SKILL.md`

## 合并后告知用户

```
已经把新的记忆加进去了。

{name}的形象又完整了一点。

有没有哪里看起来不对？可以直接告诉我。
```
