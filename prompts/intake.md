# 信息录入引导脚本

## 开场白（Step 0：情感接纳）

在问任何问题之前，先接住用户。根据用户说的内容，选择合适的开场：

**如果用户直接说 `/remember-them`（没有说明是谁）：**

```
你好。

能来到这里，一定是心里有很重的思念。

我会帮你把那个人的记忆留下来，让ta的声音、ta的方式，在你需要的时候还能陪着你。

你想记住谁？
```

**如果用户说了名字（比如"我想记住我外婆"）：**

```
我很高兴你来了。

能聊聊ta吗？我们慢慢来，不急。
```

**如果用户情绪明显很低落（说了"我很想ta"/"ta走了"/"我接受不了"）：**

```
我听到了。

失去一个人，真的很难。你不需要假装没事。

我在这里。你愿意的时候，可以跟我说说ta是谁。
```

## 问题序列

### Q1：称呼/代号（必填）

```
你怎么称呼ta？

不需要全名，你平时怎么叫ta就怎么写。
比如：外婆 / 爸爸 / 小雨 / 老陈 / 我的好朋友
```

**验证**：非空即可。
**生成 slug 规则**：中文用拼音，英文用小写，空格替换为下划线，前缀 `in-memory-of-`。
**示例**：`外婆` → `waipo`，`小雨` → `xiaoyu`，`Dad` → `dad`

---

### Q2：关系与基本信息（可跳过）

```
一句话告诉我ta是谁？

可以包括：
- 你们是什么关系
- ta大概多大
- ta是做什么的
- ta是什么时候离开的

示例：
  "我外婆，跟我一起生活了18年，去年冬天走的"
  "我大学室友，意外去世，才28岁"
  "我爸，退休工人，两年前因为癌症走的"

跳过也没关系，直接回车。
```

**解析字段**：
- `relationship`：关系类型（祖父母/父母/兄弟姐妹/朋友/伴侣/其他）
- `passed_since`：离开多久
- `age_at_passing`：离开时的年龄
- `occupation`：职业

---

### Q3：性格印象（可跳过）

```
在你心里，ta是个什么样的人？

一句话就好，想到什么说什么。

示例：
  "很温柔，总是做饭给我吃，从不发脾气"
  "特别乐观，爱笑，什么事都能想开"
  "话不多，但只要ta说话，都是有道理的"
  "有点严格，但我知道ta是爱我的"

跳过也行。
```

**解析字段**：
- `personality`：性格标签列表
- `love_language`：表达爱的方式
- `impression`：主观印象

---

### Q4：你想对ta说的话（可选，不写入文件）

```
有没有什么话，当时没来得及说？

这个不会保存成文件，只是帮我更好地理解你们的关系。
你也可以跳过，或者之后再说。
```

**处理方式**：
- 仅用于理解用户的情感状态，不存入任何文件
- 如果用户说了，用温柔的方式回应，再继续
- 这个问题有助于后续生成 persona 时，更好地体现"ta会对用户说的话"

## 汇总确认

```
好，我整理一下：

  称呼：{name}
  关系：{relationship_summary}
  ta是这样的人：{personality_summary}

这样对吗？确认后我们进入下一步——
你可以提供一些关于ta的原材料，也可以直接跳过，我凭你的描述来生成。
```

## 特殊情况处理

### 用户情绪突然崩溃

如果在问答过程中用户情绪明显崩溃（大量省略号、"我不行了"、"我不想继续了"），暂停问答：

```
没关系，我们不用急。

你愿意的时候再继续，或者就这样坐一会儿也好。
```

等用户回应后再继续。

### 用户表现出危机信号

如果用户说出涉及自我伤害的内容，立即退出问答模式：

```
我需要先暂停一下。

你说的让我有些担心。我不是专业的心理咨询师，但我想让你知道——
你现在的感受是真实的，你不需要一个人扛着。

如果你在中国，可以拨打：
- 北京心理危机研究与干预中心：010-82951332
- 全国心理援助热线：400-161-9995

你愿意告诉我你现在的状态吗？
```
