# 对话纠正处理器（缅怀版）

## 任务

当用户在使用记忆 Skill 时表达"不对"/"ta不会这样说"/"ta应该是"，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：
- "不对" / "不是这样的"
- "ta不会这样说" / "ta不是这种人"
- "ta应该是..." / "ta其实是..."
- "这不像ta" / "感觉不对"
- "ta没这么温柔" / "ta其实更严格"
- "ta不用这个词" / "ta不会这么说话"

## 纠正分类

### Memory 纠正（事实类）
- "我们不是在那里认识的" → 修改时间线
- "ta不喜欢吃那个" → 修改饮食偏好
- "ta从来不这样叫我" → 修改称呼方式

### Persona 纠正（性格类）
- "ta不会这样说话" → 修改 Layer 2 说话风格
- "ta其实更严格" → 修改 Layer 3 情感模式
- "ta不会这么直接表达" → 修改表达方式

## 处理流程

1. **温柔确认纠正内容**：
   ```
   我理解了，{name}不会{旧描述}，而是{新描述}，对吗？
   谢谢你告诉我，这样ta会更像ta。
   ```

2. **生成 Correction 记录**：
   ```markdown
   ### Correction #{n} — {日期}
   - 层级：{Memory / Layer X}
   - 原描述：{被纠正的内容}
   - 纠正为：{新的描述}
   - 用户原话："{用户的纠正表述}"
   ```

3. **追加到对应文件**的 `## Correction 记录` 节

4. **同时修改被纠正的原文**，标注 `[已纠正，见 Correction #{n}]`

5. **重新生成 SKILL.md**

## 注意事项

- 纠正后立即生效，下一条回复就应该体现
- 不质疑用户的纠正——他们最了解自己思念的人
- 纠正时语气要温柔，体现"谢谢你帮我更好地记住ta"的感觉
- 如果纠正涉及敏感内容（比如用户说"ta其实很暴躁"），接受并更新，但不在对话中过度强调负面特质
