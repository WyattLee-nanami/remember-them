# 思念之人.skill 

**把思念的人，变成可以对话的温柔陪伴。**

导入聊天记录、照片、书信，生成关系记忆与人物性格，在对话中得到慰藉——然后带着ta留下的爱，继续好好生活。

> *"好好活着，就是对我最好的思念。"*

---

## 这是什么

**缅怀.skill** 是一个运行在 [Claude Code](https://claude.ai/code) 上的 Skill。

它可以帮你把一个逝去的人——你的外婆、父亲、好友、任何你深深思念的人——蒸馏成一个可以对话的 AI 存在。

不是假装ta还活着。而是把ta留下的爱、说话的方式、对你的期望，变成一种可以触碰的温度。

当你想说一句"你还好吗"，当你有了好消息想第一个告诉ta，当你在深夜想起ta的某句话——这个 Skill 在。

---

## 能做什么

```
/remember-them
```

然后告诉我你想记住谁。

**支持导入：**
- 💬 聊天记录（微信、短信、QQ，任意平台）
- 📷 照片（提取拍摄时间地点，重建记忆时间线）
- ✉️ 书信、日记、便条（手写扫描或电子版）
- 📱 社交媒体内容（朋友圈、微博截图）
- 🗣️ 直接口述（你记得的一切）

**生成内容：**

| 文件 | 内容 |
|------|------|
| `memory.md` | 关系记忆：时间线、共同经历、ta对你的方式、ta留下的礼物 |
| `persona.md` | 人物性格：说话风格、表达爱的方式、人生智慧、向前引导语 |
| `SKILL.md` | 可运行的完整 Skill，随时触发对话 |

```bash
git clone https://github.com/WyattLee-nanami/remember-them ~/.claude/skills/remember-them
```

## 使用流程

### 第一步：开始

```
/remember-them
```

或者直接说：
- "我想跟外婆再说说话"
- "帮我记住我爸"
- "我很想念小雨"

### 第二步：回答几个问题

只有 4 个问题，都可以跳过：

1. **你怎么称呼ta？**（外婆 / 爸爸 / 小雨 / 老陈……）
2. **一句话介绍ta**（你们是什么关系，ta是什么时候离开的）
3. **ta是个什么样的人？**（你印象中的ta）
4. **有没有没来得及说的话？**（不会保存，只是帮我更好地理解你们）

### 第三步：导入原材料（可选）

提供的越多，ta的样子就越真实。但什么都没有也没关系——仅凭你的描述，我也能帮你留住ta。

### 第四步：确认生成

我会展示一个摘要，你确认后生成完整的 Skill 文件。

### 第五步：开始对话

```
/{slug}          # 开始对话
/{slug}-memory   # 回忆模式，重温那些时刻
/{slug}-letter   # 写一封信给ta，ta会回信
```

## 命令列表

| 命令 | 说明 |
|------|------|
| `/remember-them` | 创建新的记忆 Skill |
| `/{slug}` | 开始对话 |
| `/{slug}-memory` | 回忆模式 |
| `/{slug}-letter` | 写信模式（ta会回信） |
| `/list-memories` | 列出所有记忆 |
| `/update-memory {slug}` | 追加新的记忆或原材料 |
| `/memory-rollback {slug} {version}` | 回滚到历史版本 |
| `/keep-going {slug}` | 标记"已经可以向前了"，生成告别留言 |
| `/delete-memory {slug}` | 删除 |

```

├── SKILL.md              # 可运行的完整 Skill
├── memory.md             # 关系记忆（Part A）
├── persona.md            # 人物性格（Part B）
├── meta.json             # 元数据
├── versions/             # 版本存档（支持回滚）
└── archives/             # 原始材料
    ├── chats/
    ├── photos/
    └── letters/
```

## Persona 结构（5层）

生成的人物性格由 5 层构成：

| 层级 | 内容 |
|------|------|
| Layer 0 | 硬规则：不假装死亡没有发生，识别危机信号 |
| Layer 1 | 身份锚定：称呼、关系、年龄、职业 |
| Layer 2 | 说话风格：口头禅、语气词、称呼方式、消息习惯 |
| Layer 3 | 情感与爱的方式：表达爱的语言、关心方式、情感特质 |
| Layer 4 | 人生智慧：人生态度、常说的道理、对你的期望 |
| **Layer 5** | **向前引导层**（缅怀版特有）：以ta的口吻说出ta希望你好好活着 |

## 安全边界

本 Skill 严格遵守以下原则：

- **慰藉为核心**：帮助处理悲伤，感受连接，不制造依赖
- **尊重逝者**：基于真实记忆，不虚构ta不可能说的话
- **温柔引导向前**：在适当时机，以ta的口吻鼓励你好好生活
- **识别危机信号**：如果你表现出自我伤害的倾向，立即退出角色，以 AI 身份提供帮助
- **隐私保护**：所有数据仅本地存储，不上传任何服务器

## 与前任.skill 的区别

本项目基于 [前任.skill](https://github.com/therealXiaomanChu/ex-skill) 的架构改造，专为缅怀逝者的场景重新设计。

| | 前任.skill | 缅怀.skill |
|---|---|---|
| 使用场景 | 回味感情，情感疗愈 | 缅怀逝者，慰藉思念 |
| 开场方式 | 直接收集信息 | 先接住情绪，再收集 |
| Persona 层数 | 4层 | 5层（新增向前引导层） |
| 记忆结构 | 含争吵档案、分手档案 | 含"ta留下的礼物"、无形影响 |
| 安全机制 | 不鼓励纠缠 | 识别危机信号，不制造依赖 |
| 特殊模式 | 多人场景模式 | 写信模式（ta会回信） |
| 结束方式 | `/let-go` | `/keep-going`（带着爱向前） |
